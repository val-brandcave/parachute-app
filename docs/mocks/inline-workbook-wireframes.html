Callouts — screen 1

  1. 1Section hover chrome — teal outline + floating actions (Edit / Hide / Duplicate / Delete) + ⠿ drag-reorder handle. One system for every section; only the actions vary by type.
  2. 2Repeater row tools — hover a comp-grid row → edit / delete that row; "＋ Add row" under the table. Jeff's "delete Comparable 5" in two clicks, table never breaks.
  3. 3Finding block, undecided — the finding lives IN the document with Concur / Edit / Reject / Delete inline. Below it: a decided one (green record + Undo) — decisions apply instantly.
  4. 4Reviewer-added finding — first-class block, teal spine + "Reviewer-added" pip so it's never confused with AI output.
  5. 5Citation chip — every finding cites the source ("p.47 ↗"); click → Source tab scrolled to the highlighted span (screen 3).
  6. 6Add affordance between sections — "＋ Add section / ＋ Add finding" as distinct choices (content vs. first-class finding object).
  7. 7Customize ▸ closed by default — the whole 20% lives behind this one quiet button (screen 2). Demos never open it.
  8. 8Save as template — captures structure + theme (not content) so next run comes out their way. Activity = full audit ledger (screen 4).
  9. 9Prose edit-in-place — click text → caret, type, ✓/blur commits. No toolbar, no formatting — crayons.

Callouts — screen 3

  1. 1Document locked — the appraisal itself is never annotated, disputed, or edited; the doc pane carries no edit affordances. Only the finding objects are actionable.
  2. 2Deep-link target — clicking "Cited · p.47 ↗" on a workbook finding lands here, scrolled to the exact highlighted span (F-140/141 anchor work reused).
  3. 3Rail = full parity — the same Concur / Edit / Reject (+ delete, comment, flag) as the workbook blocks, same shared state: the second item shows a decision made earlier in the Workbook. Edit expands the item in place — same editor as the workbook block, no side panel.
  4. 4Create finding from a span — select source text → a finding is created with the citation pre-filled; it appears in the Workbook (and this rail) tagged "Reviewer-added." Evidence capture, not source annotation.
